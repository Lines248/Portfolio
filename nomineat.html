Context & Objective

A full-stack capstone project engineered to solve group indecision through real-time restaurant discovery and collaborative voting. The goal was to build a secure, decoupled system that bridges external API data with a custom relational database.

The Engineering Solution

  • Backend Architecture: Developed a RESTful API with Java and Spring Boot, utilizing the DAO pattern to manage PostgreSQL data persistence.
  • API Integration: Implemented a service layer to interface with the Yelp Fusion API, handling real-time data fetching and JSON parsing.
  • Frontend Reactivity: Built a dynamic UI with Vue.js to handle complex user states during the voting and selection process.

Technical Flexes

Relational Database Mapping

PostgreSQL schema and DAOs for users, restaurants, and voting transactions.
